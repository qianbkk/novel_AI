核心技术栈

FastAPI 0.115 React 18 + Vite LangGraph 1.0 SQLAlchemy 2.0 SqliteSaver 3.1 httpx tenacity sse-starlette

最聪明的设计决策

用 BackgroundTasks + asyncio.to_thread 把 LangGraph 跑在 FastAPI 进程内，告别子进程和物理 .env 覆写。LLM 路由走 DB（role_assignments 表），用户在前端页面点几下就能给任意 Agent 切模型，零代码修改。

入口

Web UI: http://localhost:5293，后端 http://localhost:8123/docs。CLI 一键 MVP: python -m scripts.run_mvp <project_id>。

状态持久化

LangGraph 用 SqliteSaver 按 thread_id=project_id 自动 checkpoint；写作引擎的状态 JSON 落 novel_AI/output/orchestrator_state.json。

人工介入

章节重写 3 次仍不达标 → human_escalation 节点 → POST /bridge/review 端点处理（accept/reject/edit）。

端到端时序（POST /bridge/run → SSE 流）

用户在前端点"写10章" → POST /projects/{id}/bridge/run → 写 BridgeRun 行（status=pending）→ BackgroundTasks.add_task(_run_bridge_async) → 拿 per-project asyncio.Lock → asyncio.to_thread(run_graph_task) 跑在默认线程池 → graph 构建 + graph.invoke(state) → _NodeWrapper 在每个 node 进出 push node_start/node_end 到 queue.Queue → SSE consumer 用 asyncio.to_thread(queue.get) 推送事件到前端。

记忆系统

两层 checkpoint + novel_AI 三层 memory 协同

核心问题

AI 上下文窗口有限（约 20 万 token）。写完 1500 章后，全塞进去一是塞不下、二是极贵。记忆系统用最小 token 数（约 1500）给 Writer 最相关的历史。novel_AI 项目加了一层 LangGraph SqliteSaver 持久化 state，让"中断-恢复"无缝。

各层说明（novel_AI 三层）

L2 热层
近 ~20 章的完整摘要
保留最近约 20 章的详细章节摘要，直接注入 Writer 的提示词。完成新章节后，最早章节自动降级到冷层。
L2 冷层
更早历史的高度压缩版本
第 20 章以前，被 Summarizer 压缩成简短摘要。只保留关键事件、人物状态变化、伏笔等核心信息，按需抽取部分注入。
L5 弧摘要
每完成一个故事弧时生成
当一个故事弧（20-40 章）写完后，整个弧压缩成一段精炼摘要。永久保留，让 Writer 了解宏观故事走向。

SqliteSaver state checkpoint

文件位置
backend/data/checkpoints.sqlite（绝对路径，避免 cwd 影响）
thread_id
project_id（每个项目独立 checkpoint 命名空间）
写入时机
每个 LangGraph node 执行后自动 checkpoint（含 state diff）
resume 流程
load_state_for_project() 优先读 SqliteSaver，回退到 novel_AI/output/orchestrator_state.json，最后 fallback 到 create_initial_state()

模型路由

DB 驱动 + 模块全局注入，6 个 LLM provider × 15 个 role

核心理念

用户在前端 /settings/providers 添加供应商账号 + API key，/settings/roles 把 15 个角色绑到不同 provider。前端点"保存" → 写 DB → 引擎 build 时 LLMRouter.install() 读 DB → 直接改 api_client 的模块全局 (ANTHROPIC_API_KEY 等)。

前端页面（React Router）

/settings/providers

Provider CRUD。6 种类型，每种 1 行；删前级联清空 role_assignments.provider_id。

/settings/roles

15 行 RoleAssignment 表格。Provider 下拉 + model_override 输入框，PUT 即时生效（下次 run 重新 install）。

/projects/:id/bridge

Bridge Console。按钮组（推送设定/生成设定包/黄金三章/写N章/状态/看板/预算/待审核）+ 滚动日志 + 当前节点状态条（Spec C）+ 人工审核卡片。

/projects/:id/worldbuild

10 阶段世界构建。SSE 流式进度，结果写入 WorldSetting/Character/Faction/PowerSystem/MapNode/Foreshadowing/Currency 等表。

/projects/:id/chapters

章节列表 + 创建 + 语义检索（按角色过滤候选章节）。导入章节时自动 embed。

/ Dashboard

项目列表 + 创建/状态概览。

CLI 脚本（backend/scripts/run_mvp.py）

python -m scripts.run_mvp <project_id> [--chapters 1] [--select A] [--skip-bootstrap]
一键跑完整 MVP 流水线：push-concept → planner → pull-setting → bootstrap → select → run N → import-chapters。流式打印 SSE 日志 + node 事件，结束给摘要。
