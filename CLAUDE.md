# CLAUDE.md — 系统级指令与行为约束

本文件是 Claude Code 在此项目中的**行为宪法**。每次对话开始时必须读取，任何操作前必须遵守以下约束。

---

## 一、项目元文件体系

本项目维护五个元文件，职责严格分离，禁止内容交叉写入：

| 文件 | 职责 | 禁止写入的内容 |
|------|------|----------------|
| `CLAUDE.md`（本文件） | Claude Code 行为规则、更新约束 | 架构介绍、使用命令、文件列表 |
| `README.md` | 项目架构、安装步骤、命令参考、功能说明 | 行为规则、文件索引、状态日志 |
| `index.md` | 所有文件的路径、类型、一句话描述、依赖关系 | 使用说明、状态信息、行为规则 |
| `PROJECT_LOG.md` | 阶段进度、关键决策、成本记录、待办事项 | 架构说明、命令列表、文件描述 |
| `.claude/lessons.md` | 已踩过的错误、根因分析、修复方式、规律总结 | 进度状态、架构说明、命令列表 |

---

## 二、元文件更新约束

**触发更新检查的条件**（每次满足以下任一条件，必须在完成主任务后检查并更新相关元文件）：

- 新增、删除或重命名任何 `.py`、`.json`、`.md` 文件
- 修改任何 Agent 的核心逻辑或接口签名
- 新增或移除 `run.py` 中的命令
- 更改 `api_client.py` 中的模型路由或新增 Provider
- 修复了一个需要超过两次尝试才解决的 Bug
- 项目阶段（Phase）发生跃迁

**各文件的更新规则**：

`index.md`：新增文件时追加条目，删除文件时移除条目，修改文件功能时更新描述行。保持表格结构不变。

`PROJECT_LOG.md`：阶段完成时更新进度表，关键决策发生时追加决策记录，成本变动时更新成本追踪表。不修改历史条目，只追加。

`README.md`：新增命令时在命令参考章节追加，架构发生结构性变化时更新架构图，安装步骤变更时同步更新。

`.claude/lessons.md`：只在修复 Bug 或踩坑后追加，格式见该文件内的模板，不修改已有条目。

## 三、代码操作规范

**修改现有文件前**，必须先用 `Read` 工具读取目标文件的完整内容，再决定如何修改，禁止凭记忆直接写入。

**新建 Python 文件前**，检查 `index.md` 确认同名文件不存在。

**运行测试**：每次修改核心模块（`agents/`、`orchestrator.py`、`memory/`、`api_client.py`）后，必须运行 `python tools/system_test.py` 确认测试全部通过，再提交结果。如测试失败，修复后再次运行，直到通过。

**禁止的操作**：
- 禁止在未读取现有内容的情况下用 `Write` 覆盖任何已存在的文件
- 禁止跳过失败的测试直接继续下一步
- 禁止在 `agents/` 目录下硬编码提示词字符串；所有提示词模板必须引用 `config/prompt_templates.py`
- 禁止在未更新 `index.md` 的情况下新增文件

## 四、调试与错误处理规范

遇到错误时，执行以下顺序：

1. 读取 `.claude/lessons.md`，检查是否有相同或相似的历史错误及已知修复方式
2. 尝试修复，最多三次独立尝试
3. 三次失败后，向用户说明：错误信息、已尝试的方案、需要用户决策的选项

修复成功后，若该错误具有复用价值（例如涉及第三方 API 的行为、LangGraph 的特定版本问题、模型输出格式问题），必须追加一条记录到 `.claude/lessons.md`。

---

## 五、对话上下文管理

当对话上下文接近上限时（通常表现为 Claude Code 提示 context window 较大），在继续工作前执行以下步骤：

1. 将当前进度追加到 `PROJECT_LOG.md`
2. 将本次对话中解决的任何新问题追加到 `.claude/lessons.md`
3. 用一段简短的「断点摘要」告知用户，内容包含：已完成的任务、下一步待做的事、任何待确认的决策
